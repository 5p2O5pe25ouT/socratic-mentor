# Socratic Mentor — Standard 档工作表

standard 档用。逐步填，每步之间停下等学习者真实回答。

## 1. 诊断

- 学习者：[谁，可能是你自己]
- 目标：[想达到什么]
- 现有水平：[新手 / 入门 / 中级 / 高级]
- 成功标准：[怎么算学会了]

诊断问题（问 3-5 个，一次一个，等答）：

1. 澄清："关于 [主题] 你已经知道什么？"
2. 应用："遇到 [简单问题] 你会怎么下手？"
3. 误解检查：[一个能暴露常见误解的问题]

填诊断结论：

- 知识缺口：
- 发现的误解：
- 起始脚手架级：[示范 / 引导 / 陪练 / 放手]

## 2. 问题阶梯（5-7 问，别硬凑）

由浅入深排序，每问标清楚目的。涉及真实代码时先读代码，用真实函数名 / 路径出题。

| # | 问题 | 目的 | 针对的缺口/误解 |
|---|------|------|----------------|
| 1 |      | 连接已有知识 | |
| 2 |      | 引出关键模式 | |
| 3 |      | 形式化 / 术语 | |
| 4 |      | 边界情况 | |
| 5 |      | 迁移到新场景 | |

## 3. 引导

按序提问。每问之后：

- 答对 → 下一问，考虑撤一格脚手架。
- 卡住 → 给一格脚手架（提示 → 类比 → 部分示例），不直接给答案。
- 答错（事实错误）→ 给反例，引导对方自己发现。

## 4. 撤脚手架

- 成功一次 → 支持少一格。
- 反复卡 2 次以上 → 加一格。
- 监控：是"有产出的挣扎"还是"挫败"。

## 5. 验证迁移

- 换一道没见过的题（近迁移）。
- 让对方用自己的话讲一遍。
- 确认起初发现的误解都消除了。
- 给下一步学习建议。
