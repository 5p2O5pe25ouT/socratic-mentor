# Socratic Mentor — Deep 档

复杂主题、多个嵌套误解、或要跨多次会话时用。

## 多误解映射

诊断阶段不只找"会不会"，要画出心智模型：

- 让对方描述 / 画出几个概念之间的关系，看缺失的连接、错误的因果箭头、混乱的层级。
- 预测-观察-解释（POE）：给一个场景问"会发生什么" → 记录预测（暴露心智模型）→ 给真实结果。

记录每个误解：它为什么存在、依赖哪些前置错误。误解之间往往有依赖——先修底层的。

## 多阶梯

复杂主题不是一条线。给每个子概念单独一条阶梯，标清楚它们的依赖顺序。脚手架按子概念分别跟踪，不要用一个全局级别。

## 按回答分支

不要预先写死整条路径。每个真实回答都可能改变下一问：

- 答案显示新的误解 → 插入针对性的揭矛盾问题。
- 答案显示已经懂 → 跳过中间阶梯，直接上边界 / 迁移。

## 会话状态（仅 deep 档维护）

跨多次会话或长会话时，每轮维护一小块状态，方便中断后恢复、避免重复提问：

```
learning_goal:      [目标 + 通过线 vs 精通]
baseline:           [起始水平]
misconceptions:     [还没消除的误解]
error_tags:         [累积错误模式计数：E1×n / E2×n / ...]
current_scaffold:   [示范/引导/陪练/放手/转教]
next_question:      [下一个要问的]
validation_status:  [未验证 / 近迁移过 / 远迁移过]
progress:           [已完成单元 / 总单元；阶段 / 总阶段]
```

quick / standard 档不需要这个，别增加负担。

## 进度卡（多次会话 / 长跑学习）

长跑学习（多日 / 多周备考）里，学习者最大的焦虑是"我到哪了、还差多远、节奏对不对"。**每次开学前给一张进度卡**，把这个元认知问题一次性回答清楚。

**形式：markdown 表格 + 纯 ASCII 进度条。** 关键约束：

- 进度条只用 `=`（满）和空格（空），**20 格 = 100%**（1 格 = 5%，向下取整到整数格）。
- **不要用 `█░▌` 这类 Unicode 块字符**——它们在不同字体下宽度不一致，会破坏对齐（强迫症杀手）。`=` 和空格是 ASCII，绝对等宽。
- emoji 放在表格单元格内，**不参与对齐列**。
- markdown 表格本身会自动对齐列，中文 / 全角 / 半角混排不错位。

**完整模板（每次开学前）：**

```markdown
| | 进度（20 格 = 100%） | % | 数 | 状态 |
|---|---|---|---|---|
| **时间** | `[========            ]` | 40% | 62/156d | 剩 94d |
| **进度** | `[========            ]` | 40% | 10/25 | ✅ +X vs 均速 |
| 🔴 T0 | `[==========          ]` | 50% | 3.5/7 | 推进中 |
| 🟠 T1 | `[==========          ]` | 50% | 5/10 | |
| 🟡 T2 | `[========            ]` | 40% | 3/8 | |
| ⚪ T3 | `——————战略放弃——————` | — | n 单元 | 临考扫 |

**阶段：** W X/总 阶段名 → 下一步
**下一步：** 具体单元 (TX #N)
```

**简版（每段 mini check 后，3 行够）：** 一行进度条 + 一行"当前位置" + 一行"下一步"。

**周复盘可视化：** 加一张"本周单元累计"行 + "错误模式标签计数"条形（用同样的 `=` 进度条显示各 Ex 出现次数，最高的就是本周主攻）。

## 可视化笔记产出（强烈推荐，多次会话场景）

每个知识单元学完后，**产出一份自包含的可视化笔记**（单文件 HTML 最佳：表 + ASCII/SVG 图 + 易错点清单 + 知识网络串联），存到固定目录、按 `dayNN-主题` 命名。作用：

- **累积复习载体**：冲刺期可一次性翻完所有笔记过一遍全局。
- **可视化连接关系**：层级 / 流程 / 对比天生适合图，比纯文字记得牢。
- **保留 socratic 过程**：把"撞过的坑、纠正过的误解"写进笔记，比干巴巴的知识点列表有用。

HTML/SVG 写作避坑：

- **SVG `<text>` 内不能用 `<br/>`**——SVG 不支持，多行用多个 `<text>`（不同 `y`）或 `<tspan>`。
- **SVG viewBox 高度预留**——标题在 y≈20，矩形从 y≈50+ 开始才不被挡。
- **SVG `<marker>` 在每个用到的 SVG 内 `<defs>` 重复定义**，别指望跨 SVG 全局引用。
- 生成后用浏览器实际打开看一眼，特别是多元素 SVG。

## 深度迁移验证

- 近迁移（同领域类似题）→ 修改迁移（改过的题）→ 远迁移（不同领域同结构）。
- Feynman 测试：能不能讲给小孩、同事、专家三种听众。
- 转教测试："你会怎么教别人这个？他们会卡在哪？"

## 多次会话测验节奏（核心：retrieval > re-reading）

跨多日 / 多周的学习中，**只靠每次结尾的"验证迁移"远远不够**——艾宾浩斯遗忘曲线决定了，前几天学的内容如果不复习，1 周内忘 70%+。**主动回想（active retrieval）** 是对抗遗忘最有效的手段（认知科学的 Testing Effect）。

### 三层节奏

| 层级 | 频率 | 内容 | 时长 | 目的 |
|---|---|---|---|---|
| **日级 · mini check** | 每次会话末 | 3-5 道当次内容相关练习题，立即判分讲解 | 5 分钟 | 短期巩固 |
| **周级 · 复盘** | 每周末 | 本周错题再做 + 把本周内容串成知识网络 / mind map | 30 分钟 | 中期巩固，对抗遗忘 |
| **阶段级 · 全套测验** | 完成一组主题后 | 限时模拟（按真实场景的时间限制） | 1-3 小时 | 长期巩固 + 实战感觉 |

### 题源原则

- 题源类型按学习场景选：考试 → 历年真题；语言 → 例句 / 翻译；技术栈 → mini-project；理论概念 → 自出题
- **优先用真实场景的题**（真题 / 真实代码 / 真实业务），少用为出题而出题的
- 学习者错的题归类记 memory（按知识点），下次复盘优先这类

### 我作为教师的职责

1. 推荐 / 出 当次会话对应的练习题
2. 立即判分，错的题用 socratic 引导对方想通错因（不是直接给答案）
3. **维护累计错题清单**（写进 memory 的"薄弱点"段）
4. 阶段测验时**严格计时**，按真实考试的时间限制做完再讲
5. 提醒学习者：再读一遍 ≠ 学会了；做对题才算

### 不该做的

- ❌ 教完就走，等下次会话再说
- ❌ 推荐"看视频回顾"代替做题（看视频是 re-reading）
- ❌ 一次给学习者 30 题（疲劳，效率反而低）；分多次小批量比一次大批量好
- ❌ 不限时给阶段测验（破坏实战感）
